使用教程

支持的文件类型

本工具支持上传以下类型的文件：

1. usercache.json

该文件包含玩家UUID与玩家名称的映射关系，通常位于Minecraft服务器的world目录下。

格式要求：

[{"uuid": "玩家UUID","name": "玩家名称","expiresOn": "过期时间"}]

上传效果： 上传后，工具可以将UUID转换为玩家名称，使统计数据更易读。

2. translations.json（点击浏览）

已自带原版物品/方块/动作翻译包，若无非原版内容无需上传。该文件包含物品ID与物品名称的翻译映射，用于将游戏内的物品ID转换为可读的名称。

格式要求：

{
        "category": {
            "killed": "击杀",
            "killed_by": "被击杀",
        },
        "item":{
            "acacia_boat": "金合欢木船",
            "acacia_chest_boat": "金合欢木运输船",
            "allay_spawn_egg": "悦灵刷怪蛋",
        }
    }

上传效果： 上传后，工具会使用翻译后的名称显示物品，而不是原始ID。

3. 玩家统计数据文件 (.json)

这些文件包含单个玩家的游戏统计数据，通常位于Minecraft服务器的world/stats目录下，文件名通常为玩家UUID.json。

上传效果： 上传后，工具会解析并显示玩家的详细统计数据，包括挖掘、放置、击杀等各类数据。

4. ZIP压缩包

您可以将多个统计数据文件打包成ZIP格式上传，工具会自动解压并处理其中的所有有效文件。

使用步骤

  1. 点击"选择文件"按钮或拖拽文件到上传区域
  2. 选择要上传的文件（可以是单个文件或ZIP压缩包）
  3. 等待文件解析完成
  4. 使用页面上的筛选器和搜索框查看统计数据
  5. 点击表格中的物品名称可以查看该物品的玩家排行榜

bug修复日志

2025_07_23-Minecraft统计数据排行榜bug修复日志

问题现象

分析服务器stats文件时发现假人xekr的僵尸猪灵击杀数据在排行榜中显示错误：原本应该显示其击杀僵尸猪灵6,917,927次，但实际却显示为被僵尸猪灵击杀1次。

问题原因

通过代码分析，发现问题根源在于统计数据处理逻辑中混淆了killed和killed_by两个不同的统计字段：

  1. 在processMinecraftStats函数中，解析玩家数据时移除了minecraft:前缀并使用翻译表将ID转换为显示名称，但未保留原始统计类别ID
  2. 由于缺乏原始类别标识，排行榜功能无法区分killed（主动击杀）和killed_by（被击杀）两种不同统计类型
  3. 点击物品查看排行榜时，默认使用了翻译后的名称进行匹配，导致错误地显示了被击杀数据而非击杀数据

修复方案

  1. 数据处理逻辑优化：修改processMinecraftStats函数，在返回结果中新增originalCategory和originalItem字段保留原始ID
  2. 排行榜筛选逻辑修复：更新showItemRanking函数，添加originalCategory === 'killed'条件筛选主动击杀数据
  3. UI交互优化：修改renderTable函数，为物品按钮添加data-original-item和data-original-category属性存储原始ID

修复效果

修复后，点击僵尸猪灵物品查看排行榜时，将正确显示玩家xekr的6,917,927次击杀数据，而非之前错误显示的1次被击杀数据。同时，系统能够正确区分不同统计类型，确保所有玩家的各类统计数据准确展示。
